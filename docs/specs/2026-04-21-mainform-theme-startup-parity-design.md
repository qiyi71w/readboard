# MainForm 旧新版主题统一设计

日期：2026-04-21

## 背景

`MainForm` 在几轮 DPI / 分辨率兼容修补后，主题相关行为发生了两类偏移：

1. 新版主题本身已经偏离用户认可的“以前的新版本主题”样式，头部、同步区和按钮区的排版节奏都被拉宽了。
2. 默认主题与新版主题不再只是“视觉不同”，而是窗口尺寸、布局分支和控件几何都发生了分叉。

用户最终确认的目标不是继续保留两套不同视觉，而是回到截图所示的旧新版主题样式，并把默认主题和新版主题都统一成这一套样式。

## 最终决策

- 以历史提交 `c689ea9` 对应的“旧新版主题”作为唯一视觉与排版基准。
- 默认主题和新版主题都复用这套旧新版主题视觉，不再保留经典 WinForms 外观差异。
- 主布局共享 profile 只使用旧新版主题口径做文本测量，不再把经典主题字体重新灌回布局判断。
- `legacy` 桌面布局恢复旧新版主题的关键逻辑常量，再通过现有 `ScaleValue(...)` 做 DPI 缩放。
- 小窗口、高 DPI 或窄屏场景继续保留 `adaptive` 回退，避免重新引入分辨率兼容问题。
- 主题切换入口保留，仅用于配置兼容和菜单语义；切换后两种主题显示结果一致。

## 根因

这次偏移不是颜色 token 或控件皮肤本身出了问题，根因主要有两个：

### 1. 共享布局度量被“主题中和”过头

之前为了让两个主题布局一致，引入了“按经典主题和新版主题分别测量，然后取较大值”的共享测量方式。  
这一步虽然降低了主题切换时的分叉概率，但也把旧新版主题原本较紧凑的节奏拉坏了，直接表现为：

- 头部平台区和右侧工具列之间被拉出过大的空白
- 同步区的列宽比旧新版主题更松
- 旧新版主题原有的按钮排布和视觉密度被破坏

### 2. 默认主题仍然保留独立的经典视觉路径

即使布局测量被统一，只要默认主题仍走经典 WinForms 字体、按钮和边框路径，运行时的 `PreferredSize`、字体渲染和按钮外观仍会继续制造新的视觉与几何差异。

## 设计原则

- 旧新版主题是唯一基准，不再做 classic/new theme 折中样式。
- 布局统一优先于“保留两套不同皮肤”。
- DPI 兼容必须保留，不能回退到纯像素硬编码。
- 小窗口自适应必须保留，不能为了还原旧样式而牺牲低分辨率可用性。

## 方案细节

### 1. 主题视觉统一

`ApplyMainFormTheme()` 统一先应用 `UiTheme.ApplyWindow(this)`，然后默认主题与新版主题都走旧新版主题的控件样式入口。

结果：

- 两个主题的背景、按钮、输入框、单选/复选框、分组框视觉一致
- 不再存在默认主题切回系统按钮样式后引发的布局与观感漂移

### 2. 共享布局 profile 改为旧新版主题单一口径

`MainFormLayoutProfile` 继续保留，但只按 `UiTheme.BodyFont` 进行按钮、选项和标签测量。

不再允许：

- `Control.DefaultFont`
- `Math.Max(classicWidth, optimizedWidth)`
- 任何根据当前主题即时字体状态决定主布局分支的逻辑

### 3. 恢复旧新版主题的 legacy 几何节奏

在桌面宽度足够时，`legacy` 布局恢复旧新版主题的关键逻辑常量，并在必要处仅把它们作为下限：

- 头部按钮宽度基准：`72 / 68 / 68`
- 头部右侧工具列宽度基准：三按钮行宽
- 棋盘规格输入框基准：`34 x 24`
- 同步区关键槽位基准：
  - 自动落子条件槽：`137`
  - 计算量标签槽：`112`
  - 时间标签槽：`61`
- 操作按钮区继续沿用旧新版主题两行节奏，并保留当前产品新增的 `btnForceRebuild`

这些常量不再被“共享最大值”直接放大；只有当当前语言文本确实更长时，才在不破坏结构的前提下向上扩展。

### 4. 自适应布局继续保留

当当前 `ClientSize` 无法容纳旧新版主题桌面布局时，仍回退到 `adaptive` 分支：

- 允许头部平台选项换行
- 允许同步区 `FlowLayoutPanel` 包裹
- 允许底部按钮自动折行

但自适应布局仍使用同一套旧新版主题测量口径和槽位下限，避免再次出现“adaptive 虽然能用，但已经不像旧新版主题”的情况。

### 5. DPI / 分辨率兼容策略

- 所有旧新版主题几何常量都通过 `ScaleValue(...)` 参与 DPI 缩放
- `ClientSize` 仍受当前工作区约束，避免超出屏幕
- 内容超高时继续使用 `AutoScroll`
- 布局分支判断只依赖共享 profile 与当前可用宽度，不依赖主题状态

## 实现范围

本次设计覆盖：

- `readboard/Form1.cs`
- `readboard/MainForm.LayoutProfile.cs`
- 与主题 / 主布局相关的 Host 回归测试
- 本文档和对应 implementation plan

不覆盖：

- 其他窗体的主题体系重构
- 配置格式调整
- 发布脚本或宿主集成改造

## 验收标准

- 默认主题和新版主题启动后显示相同的旧新版主题视觉样式
- 默认主题和新版主题在同一 DPI / 同一 `ClientSize` 下命中相同布局
- 平台区、棋盘规格区、同步区、按钮区回到旧新版主题节奏
- `最大计算量(选填)` 与 `首位/首选计算量(选填)` 保持对齐
- 高 DPI、不同分辨率和窄屏场景下仍能稳定回退到自适应布局
