# readboard 旧版桌面布局恢复设计

日期：2026-04-21

## 目标

在不回退现有分辨率 / DPI 兼容修复的前提下，让以下窗体恢复接近旧版桌面布局的视觉秩序：

- `MainForm`
- `SettingsForm`
- `TipsForm`
- `FormUpdate`

核心要求：

- 正常桌面分辨率与常见 DPI（100% / 125% / 150%）下，默认优先呈现旧版桌面骨架。
- 只有在宽度、高度或 DPI 放大导致空间确实不足时，才触发局部退化。
- 退化优先级是“局部堆叠 / 局部换行 / 垂直滚动”，不是“默认流式重排”。

## 布局原则

### MainForm

- 默认恢复旧版两列头部结构：
  - 左侧平台类型分组。
  - 右侧工具按钮列。
- 棋盘规格区默认保持旧版紧凑单行结构。
- 同步区默认保持旧版两排结构，不让 `FlowLayoutPanel` 在正常桌面上默认换行。
- 操作区默认保持旧版两排按钮节奏。
- 当宽度不足以容纳旧版骨架时：
  - 先让头部右侧工具列下移。
  - 再让操作按钮按原分组换到下一排。
  - 高度仍不足时，启用垂直滚动兜底。

### SettingsForm

- 默认恢复旧版三行选项区。
- 默认恢复旧版两列字段区。
- 默认恢复旧版底部按钮单行布局。
- 当宽度不足时：
  - 先让某一行双选项改为纵向堆叠。
  - 再让字段区从双列降为单列。
  - 高度不足时，只启用滚动，不改变字段逻辑顺序。

### TipsForm

- 默认保持旧版文本布局与右下角横向双按钮。
- 当宽度不足以容纳双按钮时，再改为上下堆叠。
- 高度不足时启用滚动兜底。

### FormUpdate

- 默认保持现有更新弹窗的信息层级和桌面比例。
- 按钮默认保持右下角横排。
- 当空间不足时：
  - 优先压缩可视文本区高度。
  - 再把整体窗口限制在屏幕工作区内。
  - 不改变信息区顺序。

## 实现约束

- 不修改业务功能、协议、配置语义。
- 只修改布局策略与尺寸约束逻辑。
- 保留已有 DPI 感知、`WM_DPICHANGED` 响应、屏幕工作区约束与滚动兜底。
- 继续避免把正常桌面布局做成默认流式响应。

## 验证要求

- 针对 `Form1.cs`、`Form4.cs`、`Form7.cs`、`FormUpdate.cs` 补充或更新源码回归测试。
- 保留高 DPI 回归测试中的滚动兜底与高度钳制断言。
- 运行 `Readboard.VerificationTests` 相关布局 / 高 DPI / 启动回归验证。
